Recognizing limitations of static negative sampling, Asynchronous Negative Contrastive Esti- mation (ANCE) [266] addressed this by refreshing hard negatives during training. RocketQA [203] improved contrastive training with cross-batch memory for more diverse negative samples. Gen- eralizable T5 Retriever (GTR) [181] framed retrieval as a sequence- to-sequence task rather than solely relying on contrastive learning. Efficient and Scalable Dense Retrieval. Another research line focused on scalability and efficiency. ColBERT [113] introduced late interaction mechanisms, preserving token-level representa- tions to reduce memory overhead. CoCondenser [62] optimized intermediate representation learning for more compact document embeddings. E5 [246] explored improved contrastive pretraining techniques for adaptability to various tasks. Impact. Dense Retrieval models [111, 113, 181, 246, 266] offer sig- nificant advantages over Sparse Retrieval by enabling semantic matching when exact lexical overlap is absent. However, Dense Retrieval is limited by dependency on large-scale labeled training data [54, 131, 304] and lack of interpretability [31, 105, 148]. Retrieval with Edit Distance. Edit distance-based Retrieval matches queries and documents by comparing edit distances between words and phrases like Levenshtein Distance [69, 77, 282]. These methods are especially popular in code retrieval [74], often integrated with abstract syntax trees of code snippets [200, 287]. Retrieval with Named Entity Recognition. Named Entity Recog- nition (NER) [130, 177, 270]-based retrieval uses entities in queries and documents for matching, allowing text to be understood at a concept level [141, 268]. Entity-based retrieval enforces type matching, especially useful for specific scenarios like legal do- main [49, 121]. Retrieval with Large Language Models. Recently, Generative Retrieval (GR) [13, 41, 231, 236, 252] has emerged as a new par- adigm leveraging generative model [2, 206, 239] parameters for memorizing documents, enabling retrieval by directly generating relevant document identifiers. Given the same parameter size and training data, GR can achieve better performance [236, 252] and interpretability [236, 290], becoming a new research focus [134]. 3.2 Recent Retrieval Strategies While building on major retrieval frameworks discussed above, diverse strategies have been recently introduced at various stages of the retrieval pipeline. 3.2.1 Hybrid Retrieval. Hybrid retrieval [63, 150, 162] combines dif- ferent retrieval models to leverage complementary strengths. Gao et al. [63] proposed CLEAR (Complementing Lexical Retrieval with Semantic Residual), learning a dense "residual" vector to comple- ment BM25. Luan et al. [150] explicitly combined sparse and dense representations, achieving better ranking accuracy than strong single-method baselines. This fused retrieval fashion enhances ef- fectiveness and robustness [241, 286]. 3.2.2 Data Augmentation for Retrieval. To enlarge training scale, data augmentation [15, 21, 183, 184] leverages additional data be- yond original labeled queries and documents. Doc2query [184] trained a transformer [242] to predict likely queries for each docu- ment. DocT5query [183] used T5 [206] to generate multiple queries per document.

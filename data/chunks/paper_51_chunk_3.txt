Generation Retrieval-Augmented Generation (RAG) enhances LLMs by retriev- ing relevant information from external sources before generating responses [124]. RAG addresses hallucination, outdated knowledge, and domain expertise limitations by grounding responses in re- trieved documents [82, 86]. The RAG pipeline includes indexing (document processing/vectorization), retrieval (identifying relevant information), and generation (with retrieved context) [16, 64]. 2.2.1 RAG Paradigms. (1) Naive RAG. This initial approach em- ploys a straightforward "retrieve-read" framework [159], where documents are retrieved based on the original query and directly fed into the model. While effective, this approach struggles with retrieval precision and may introduce irrelevant information that can mislead LLMs [24, 95, 261]. (2) Advanced RAG. This para- digm introduces pre-retrieval and post-retrieval optimizations. Pre- retrieval techniques focus on improving indexing structures and query formulation through query rewriting [159], query expansion [198], and hybrid retrieval approaches [63]. Post-retrieval processes enhance integration through reranking and context compression techniques [14], significantly improving response relevance. The most recent paradigm represents a flexible ar- chitecture incorporating specialized components for specific tasks [251]. This approach introduces functional modules like search in- terfaces, memory systems, and task adapters, enabling sophisticated retrieval strategies [114] and patterns like rewrite-retrieve-read [159], generate-read [281], and demonstrate-search-predict [112]. 2.2.2 Key Components and Techniques. (1) Retrieval Sources. While unstructured text data remains predominant [249], research explores semi-structured data [133], structured knowledge graphs [106, 251], and even LLM-generated content [30, 234]. (2) Retrieval Granularity and Embeddings. Granularity ranges from fine- grained (tokens, sentences) to coarse-grained (chunks, documents) units [28], each balancing information completeness against re- trieval precision. Embedding models have seen substantial improve- ments through specialized architectures [135, 185] and alignment fine-tuning [40, 291]. (3) Context Processing. Reranking meth- ods [315] prioritize the most relevant documents while context compression techniques [267] reduce redundancy while preserving essential content, addressing the "lost in the middle" problem [144]. 2.2.3 Advanced Workflows and Integration. RAG has evolved be- yond static pipelines to include adaptive retrieval [6, 98], where models autonomously determine when retrieval is necessary; it- erative retrieval [221], which employs recursive cycles to refine responses; and recursive retrieval [114], which hierarchically de- composes complex queries. Despite increasing context windows in modern LLMs, RAG maintains importance through efficiency, transparency, and domain adaptability [194, 262]. The approach continues to evolve through integration with fine-tuning [140, 154], creating hybrid systems. Recent evaluations reveal that while fine- tuning shows some improvement, RAG consistently outperforms it for both existing and new knowledge [193].
